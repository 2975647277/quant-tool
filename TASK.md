# A股量化交易产品设计任务

## 1. 项目目标

构建一个面向 A 股的本地量化研究与决策辅助工具，并以东方财富 macOS 桌面 App 的伴随侧栏作为第一使用入口。

目标应用：

- 应用名称：东方财富；
- macOS Bundle ID：`com.eastmoney.explore`；
- 集成方式：通过 macOS Accessibility API 只读识别当前股票和窗口状态；
- 展示方式：独立本地侧栏贴靠东方财富窗口，不向东方财富进程注入代码。

核心使用场景：

> 用户在东方财富 App 中切换股票后，本地侧栏在 1 秒内自动显示该股票的量化评分、未来 10 日相对收益排名、风险区间、评分解释、数据时间和模型版本。

量化系统形成以下闭环：

> 高质量数据 → 可验证因子 → 市场状态识别 → 股票排序 → 组合构建 → 风险控制 → 模拟盘反馈

产品不追求宣传式的“明日涨跌准确率”，而是在明确持有周期和风险约束下，提高选股排序能力、盈亏比和组合收益稳定性。

第一版成功标准：

- 连续切换 50 只股票时，股票代码识别成功率不低于 95%；
- 已缓存股票的诊断结果在 1 秒内完成展示；
- 东方财富升级或识别失败时，可手动输入股票代码继续使用；
- 全部核心数据、模型和用户配置默认保存在本机；
- 不读取账号密码、交易凭证或其他非必要敏感信息；
- 不执行自动买卖或其他资金操作；
- 模型必须提供样本外成绩、评分依据和失效条件。

## 2. 产品定位

第一阶段定位为东方财富桌面 App 的本地量化伴随工具，主要服务具有一定交易经验的个人投资者。

产品形态不是东方财富官方插件，也不是对东方财富客户端进行逆向修改，而是一个独立安装、独立运行、能够跟随东方财富当前股票的本地应用。

核心能力：

- 自动识别东方财富当前查看的 A 股股票；
- 以贴靠侧栏、悬浮窗或快捷键面板展示结果；
- 识别当前市场状态、风险温度和主导风格；
- 展示行业强弱和市场宽度；
- 对全 A 股票进行多维度评分和横截面排序；
- 预测未来 5、10、20 个交易日的相对收益；
- 给出预期收益、风险区间和模型置信度；
- 解释评分来源及策略失效条件；
- 生成并跟踪模拟组合；
- 公开展示模型样本外成绩和失效阶段。

暂不包含：

- 注入、Hook、逆向或修改东方财富 App；
- 批量抓取东方财富界面数据作为模型训练数据；
- 读取东方财富登录凭证、持仓和交易信息；
- 自动实盘下单；
- 高频或逐笔交易策略；
- 保证收益、必涨股票或指令式荐股；
- 未经验证的深度学习价格预测。

## 3. 第一版策略

实现一个全 A 股中低频多因子选股模型：

- 调仓频率：每周；
- 持仓数量：10～20 只；
- 预测周期：未来 10～20 个交易日；
- 预测目标：相对行业或宽基指数的超额收益排序；
- 基线模型：因子模型 + LightGBM/XGBoost；
- 组合约束：行业、市值、流动性、波动率、单股权重和换手率；
- 运行方式：只生成研究结果与模拟组合。

## 4. MVP 功能

### 4.1 东方财富联动侧栏

实现：

- 检测东方财富启动、退出、激活和窗口移动；
- 使用 `AXUIElement` 和 `AXObserver` 只读获取当前股票名称与代码；
- 股票变化后自动刷新量化诊断；
- 侧栏可自动贴靠、取消贴靠、置顶和隐藏；
- 提供全局快捷键和菜单栏入口；
- 识别失败时允许手动输入股票代码；
- 东方财富未运行时可以作为独立量化工具使用。

### 4.2 市场状态

展示：

- 市场状态：上涨、震荡偏强、震荡偏弱、下跌或高波动；
- 风险等级；
- 主导风格；
- 行业强弱；
- 市场成交活跃度和宽度；
- 建议研究仓位区间。

### 4.3 量化选股

支持按以下条件筛选：

- 持有周期；
- 最大可接受回撤；
- 市值范围；
- 行业和风格；
- 最低流动性；
- 是否排除 ST、新股、停牌股和财报期；
- 最大股票数量。

输出股票的超额收益排名、上涨概率、风险评级和模型置信度。

### 4.4 个股诊断

至少展示：

- 综合评分及各维度原始分；
- 未来 10 日跑赢行业概率；
- 预期收益及风险区间；
- 趋势、基本面、估值、资金和风险解释；
- 模型失效条件；
- 数据更新时间和模型版本。

侧栏默认使用精简视图，突出综合评分、未来 10 日排名、风险和前三条解释；展开后显示完整因子明细和历史表现。

### 4.5 模拟组合

生成保守、均衡和进取三类模拟组合，并展示：

- 预期收益与波动；
- 最大历史回撤；
- 行业和风格暴露；
- 单股权重；
- 调仓记录；
- 交易成本；
- 模型版本。

### 4.6 模型成绩单

展示最近 30 日、90 日和 1 年的：

- 样本外胜率；
- Rank IC 和 ICIR；
- Top 组超额收益；
- 年化收益和最大回撤；
- Sharpe、Sortino 和 Calmar；
- 换手率与交易成本；
- 回测和模拟盘偏差；
- 不同市场状态下的表现。

## 5. 数据要求

### 5.1 数据范围

- 日线和 30 分钟行情；
- 成交量、成交额、换手率和涨跌停状态；
- 指数、行业和风格行情；
- 财务报表及实际披露时间；
- PE、PB、PS、股息率等估值指标；
- 业绩预告、回购、减持、解禁、问询等事件；
- 市场宽度、波动率、风格强弱和拥挤度。

### 5.2 数据质量

必须避免：

- 未来函数；
- 幸存者偏差；
- 错误复权；
- 忽略停牌和涨跌停；
- 忽略流动性；
- 忽略交易成本。

财务和事件数据必须按实际可获得时间保存，支持 Point-in-Time 查询和数据版本追踪。

## 6. 因子与模型

第一版建设 30～50 个核心因子，后续扩展到 50～100 个。

因子分类：

- 趋势与动量；
- 价值；
- 质量；
- 成长；
- 交易与资金；
- 风险。

每个因子必须记录：

- 定义和数据来源；
- 更新时间和有效范围；
- 缺失值处理；
- 行业及市值中性化方法；
- 历史 IC、ICIR 和换手率；
- 衰减周期和拥挤度。

模型输出：

- 未来 5、10、20 日超额收益分数；
- 横截面排名；
- 上涨及跑赢行业概率；
- 未来最大回撤概率；
- 风险事件概率。

## 7. 股票池与交易约束

股票池应排除或单独处理：

- ST 和退市整理股票；
- 上市时间过短的新股；
- 长期停牌股票；
- 成交额过低股票；
- 一字涨跌停股票；
- 财务异常及重大风险事件股票。

回测必须模拟：

- T+1；
- 涨跌停和停牌；
- 佣金、印花税和过户费；
- 滑点、冲击成本和开盘偏离；
- 成交量容量；
- 调仓延迟。

## 8. 验证标准

不得随机拆分时间序列数据。采用 Walk-Forward 验证，并根据标签重叠情况加入 Purging 和 Embargo。

内部 MVP 准入参考：

- 样本外年化 Sharpe 不低于 0.8；
- 样本外最大回撤不高于 15%；
- Top 组长期取得正超额收益；
- 月度跑赢基准比例不低于 55%；
- 单一行业暴露不高于 25%；
- 单只股票权重不高于 8%；
- 结果不依赖单一年份或单一行业；
- 回测与模拟盘偏差可解释。

上述数值是研发验收标准，不构成收益承诺。

## 9. 技术架构

### 9.1 桌面端

- Tauri 2；
- Rust；
- Vue 3；
- TypeScript；
- Vite；
- Element Plus；
- ECharts；
- Pinia；
- Vitest。

Rust 层负责：

- 东方财富进程和窗口发现；
- macOS Accessibility 权限检测；
- `AXUIElement` 界面读取；
- `AXObserver` 股票和窗口变化监听；
- 侧栏贴靠、置顶、隐藏和快捷键；
- 本地 Python 服务生命周期管理。

Vue 层只负责展示和交互，不直接访问东方财富或系统敏感接口。

### 9.2 本地量化服务

- Python 3.12；
- FastAPI；
- Pydantic；
- Polars；
- NumPy；
- LightGBM；
- scikit-learn；
- Pytest。

服务只监听 `127.0.0.1`，使用随机端口和会话令牌；第一版开发期由 Tauri 启动独立 FastAPI 进程，发布时打包为 Python sidecar。

### 9.3 本地数据层

- DuckDB；
- Parquet；
- SQLite。

用途：

- Parquet：日线、财务、事件、因子和预测结果；
- DuckDB：本地分析查询、聚合和回测数据读取；
- SQLite：应用配置、模型注册、任务状态、诊断历史和模拟组合。

第一版不使用 PostgreSQL、Redis、MinIO、ClickHouse 和微服务，因为产品是单机、单用户、日频批处理场景。未来增加云同步、团队协作或大规模分钟数据后再引入服务端基础设施。

### 9.4 模型与验证

- 第一基线：规则评分和线性因子组合；
- 第一正式模型：按交易日分组的 LightGBM Ranker；
- 第一标签：未来 10 日相对行业超额收益；
- 验证方式：Walk-Forward，并设置至少 10 个交易日的隔离区；
- 解释方式：原始因子贡献、分位数和模型特征贡献；
- 调参工具：模型稳定后再引入 Optuna。

### 9.5 数据源适配

定义统一的 `MarketDataProvider` 接口，支持：

- 用户导入 CSV/Parquet；
- 经授权的数据 API；
- 用户具备相应服务权限时接入 Choice 量化接口；
- 本地模拟数据，用于开发和演示。

不得将东方财富 App 界面或未授权接口作为批量训练数据源。数据提供方、授权范围、更新时间和版本必须随数据记录。

## 11. 合规边界

第一阶段只提供研究、统计、因子分析、风险指标、自定义筛选、回测和模拟组合。

桌面联动必须遵守：

- 不注入、Hook、反编译或修改东方财富 App；
- 不绕过登录、付费、访问控制或安全机制；
- 不读取或保存账号密码、验证码、交易凭证和资金信息；
- 不批量复制东方财富行情数据用于模型训练或商业分发；
- Accessibility API 默认只读，只获取完成股票识别所需的最小信息；
- 不模拟点击买入、卖出、确认委托等交易操作；
- 产品名称和页面必须明确声明与东方财富不存在官方隶属或授权关系。

禁止使用以下表达或功能：

- 保证收益；
- 今日必买；
- 明天涨停；
- 内部票；
- 跟着买；
- 未经许可的自动下单或明确指令式买卖建议。

后续若提供收费的具体证券建议、投资组合或自动交易能力，必须先由证券业务律师评估，并优先考虑与持牌券商或投顾机构合作。

## 13. 产品想法与优先级

### 13.1 第一版必须验证

- 股票切换后侧栏是否真正节省用户操作；
- 精简评分卡是否能在 10 秒内被理解；
- 用户更重视收益排名、风险提醒还是评分解释；
- 用户是否愿意为稳定的数据和模型成绩单付费；
- 东方财富版本升级后股票识别是否仍然可靠。

### 13.2 后续候选功能

- 自选股批量诊断和风险排序；
- 两只股票横向对比；
- 本地投资笔记与量化结果关联；
- 事件日历和财报、解禁、减持提醒；
- 模型信号变化记录；
- 模拟组合与东方财富当前股票联动；
- 一键导出诊断报告；
- Windows 版东方财富伴随窗口；
- 经用户授权的 Choice 数据适配器。

### 13.3 明确不做

- 自动点击或提交订单；
- 读取账户资产和真实持仓；
- 通过逆向接口抓取受限数据；
- 以“AI 必涨”“高胜率荐股”作为营销承诺；
- 在基础数据和回测未验证前堆叠深度学习模型。
