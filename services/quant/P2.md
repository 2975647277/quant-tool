# P2 真实数据与因子层

## 当前实现

P2 建立了统一 `MarketDataProvider`、原始层、清洗层、因子层和模型工件层，并将真实数据重新送入 P3。

默认在线适配器的数据范围：

- 中证指数网站：当前沪深 300 成分；
- 新浪财经：A 股未复权/后复权日线和沪深 300 指数日线；
- 新浪财经财务报表：资产负债表、利润表、现金流量表；
- 财务字段保留报告日、公告日期和更新日期。

在线数据仅用于本地非商业研究。商业发布前必须确认各数据提供方授权，或实现 Choice、Wind、Tushare 等用户已有权限的数据适配器。

## Point-in-Time 规则

财务记录最早可用时间定义为：

```text
available_at = max(公告日期 00:00, 数据更新时间)
```

某个交易日只允许读取 `available_at <= 当日收盘` 且 `报告日 <= 当日` 的最新报表。当前接口没有提供完整的历史修订链，因此旧报表如果后来被修订，会按修订时间才生效。这种方法偏保守，但不会把今天看到的修订值提前放进历史。

每次更新都会保存：

- provider ID；
- 原始来源页面；
- 用途范围；
- 抓取时间；
- 数据版本；
- Point-in-Time 时间；
- 数据质量警告。

## 15 个首批因子

价格和交易因子：

1. 5 日动量；
2. 20 日动量；
3. 60 日动量；
4. 5 日反转；
5. 20 日年化波动率；
6. 20 日下行波动率；
7. 60 日最大回撤；
8. 距 60 日高点；
9. 20 日平均换手率；
10. 5 日/20 日换手率变化；
11. 20 日平均成交额对数；
12. 20 日 Amihud 非流动性。

Point-in-Time 财务因子：

13. ROE；
14. 资产负债率；
15. 同报告期营业收入同比增长。

原始因子按交易日做 1%/99% 缩尾、横截面中位数填充和 Z-Score。若某个交易日任一因子缺失比例超过 40%，或横截面少于 20 只股票，整日数据会被拒绝，而不是静默进入模型。

## 运行

```bash
pnpm research:p2
```

可缩短范围进行联调：

```bash
services/quant/.venv/bin/python services/quant/scripts/run_p2_real.py \
  --start 2022-01-01 \
  --end 2025-12-31 \
  --universe-size 30
```

默认结束日期为执行当天，数据源只返回已经完成的交易日。可以强制把当前沪深 A 股纳入试运行股票池，即使它不是沪深300成分：

```bash
pnpm research:daily
```

当前 `research:daily` 为本机试运行任务，固定要求包含 `002463`；扩大到全 A 股票池后移除该临时固定项。

结果保存在仓库的 `data/` 目录，不提交到 Git。API 只读取已经完成的本地工件，不会在界面请求中临时抓取网络：

- `GET /v1/research/p2/status`
- `GET /v1/research/p3/real`
- `GET /v1/research/p3/current`
- `GET /v1/stocks/{code}/research`：将当前股票映射为当前 Top 20、研究池内未入选或未覆盖。

有标签因子数据用于 Walk-Forward 和最终训练；最新完整交易日另外构建无标签因子截面，用于当前推理。个股研究接口不会为 30 只样本之外的股票生成排名或上涨概率。

只使用已经下载的 P2 工件重跑 P3：

```bash
pnpm research:p3:real
```

## 当前准入阻断项

这次实现可以验证真实数据到模型/回测的完整工程链，但在以下问题解决前不会注册默认模型：

- 当前沪深 300 成分回测历史，存在幸存者偏差；
- 数据源没有给出财务报表的完整历史修订链；
- 历史行业成员数据未接入，目前标签使用相对沪深 300 的未来 10 日超额收益；
- 涨跌停状态按各板块开盘价相对前收的阈值近似，尚未接入逐日 ST 状态和证券简称历史。

下一步应接入带历史成分、历史行业分类、报表修订版本和逐日证券状态的正式授权数据源，再解除这些准入门禁。
