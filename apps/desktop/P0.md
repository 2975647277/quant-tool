# P0：东方财富桌面联动验证

## 范围

P0 只验证以下能力：

1. 检测东方财富 macOS App 是否运行；
2. 通过只读 Accessibility API 识别当前股票名称和 6 位代码；
3. 获取东方财富窗口位置与尺寸；
4. 将量化侧栏贴靠到东方财富窗口；
5. 自动识别失败时允许手动输入股票代码。

P0 不采集行情、不计算因子、不训练模型、不读取账户、不执行交易。

## 本地运行

工具链：

- Node.js `24.18.0` LTS（仓库通过 `.nvmrc` 固定）；
- pnpm `8.15.4`；
- Rust `1.97.1`。

```bash
nvm use
pnpm install
pnpm dev
```

首次使用时，在侧栏中点击“请求权限”，然后在：

```text
系统设置 → 隐私与安全性 → 辅助功能
```

允许“量化伴随侧栏”访问。重新启动开发应用后即可自动识别。

构建 macOS App：

```bash
pnpm build
```

产物位于：

```text
apps/desktop/src-tauri/target/release/bundle/macos/量化伴随侧栏.app
```

## 实现说明

- 识别属性限定为 `AXTitle`、`AXValue` 和 `AXDescription`；
- 窗口属性限定为 `AXFocusedWindow`、`AXPosition` 和 `AXSize`；
- `AXObserver` 负责接收焦点、值、标题、布局和窗口变化通知；
- 每 650ms 进行一次只读轮询，作为东方财富自定义控件不发送通知时的兜底；
- 最多遍历 1,200 个元素、10 层深度，避免无界遍历；
- 当前股票同时支持“名称（6位代码）”单节点，以及东方财富实际使用的“名称”与“（6位代码）”相邻双节点结构；
- 侧栏不会调用 `AXUIElementPerformAction` 或设置任何东方财富属性。
