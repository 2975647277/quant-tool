# P1：本地诊断闭环

## 已实现范围

P1 已打通：

```text
东方财富当前股票
        ↓
Tauri / Rust 进程管理与请求代理
        ↓
127.0.0.1 随机端口 FastAPI
        ↓
确定性模拟诊断
        ↓
Vue 评分卡自动刷新
```

具体能力：

- 健康检查 `GET /health`；
- 个股诊断 `GET /v1/stocks/{code}/diagnosis`；
- Pydantic OpenAPI 协议和自动生成的 TypeScript 类型；
- 股票切换后自动加载评分卡；
- 综合分、未来 10 日超额排名、上涨概率、预期收益、下行风险；
- 趋势、基本面、估值和资金行为四个模拟维度；
- Rust 自动启动、健康检查、停止和异常重启 Python 服务；
- 服务不可用时的明确状态和手动重试；
- Ruff、Pytest、Vitest 和 Vue TypeScript 检查。

## 运行

在仓库根目录：

```bash
nvm use
pnpm install
pnpm dev
```

首次使用还需要在：

```text
系统设置 → 隐私与安全性 → 辅助功能
```

允许当前构建的“量化伴随侧栏”访问东方财富的只读界面信息。

## 本地服务边界

- 固定绑定 `127.0.0.1`，不监听局域网地址；
- 使用操作系统分配的随机空闲端口；
- 每次桌面应用启动生成新的随机会话令牌；
- 令牌通过子进程环境变量传递，不进入 WebView 和进程命令行；
- WebView 只调用 Tauri 命令，由 Rust 代理本地 HTTP 请求；
- 桌面应用正常退出时主动终止对应 Python 子进程，异常终止时 Python 也会检测父进程并自行退出；
- 服务失去响应后由 Rust 守护线程自动恢复。

## 当前数据说明

诊断结果由股票代码哈希生成，因此同一股票的评分稳定、不同股票的评分不同，便于测试 UI 与联动。它不包含任何真实行情、财务、资金或事件数据。

真实数据将在 P2 按 Point-in-Time、来源授权和版本留痕规范接入。P1 页面必须持续显示“模拟”与“不构成投资建议”。

P1 的 `.app` 仍依赖仓库内的 `services/quant/.venv`，只作为本机验证构建；独立可分发 sidecar 留到 P5。
